# 매치다(MatchDa) UX 리뷰 & 홍보 전략 — 2026-07-12

신규 유저 관점으로 전체 여정(랜딩 → 가입 → 온보딩 → 잡 탐색 → 이력서/워크스페이스 → 지원 관리)을
코드 레벨 + 라이브 사이트(matchda.com)로 점검한 결과입니다.

---

## 1. 총평 — 해외 취업 준비자에게 유용한가?

**결론: 컨셉과 핵심 기능은 확실히 유용하다. 다만 "랜딩이 약속하는 것"과 "실제 제품이 주는 것" 사이의 갭이 지금 가장 큰 리스크다.**

강점 (경쟁 서비스 Teal, Simplify, Careerflow 대비 차별점):
- **한국어로 입력하면 영어 이력서가 나온다** — 한국인 해외 취업 준비자의 진짜 페인포인트를 정확히 찌름. 온보딩 채팅("한국어로 편하게 답해주세요")은 심리적 장벽을 크게 낮추는 좋은 설계.
- **수집 → 매칭 점수 → 맞춤 이력서 → 커버레터 → 칸반 관리**가 한 흐름으로 이어짐. 보통 3~4개 툴(스프레드시트 + DeepL + ChatGPT + Notion)로 하던 일을 하나로 통합.
- Greenhouse/Lever/Ashby/SmartRecruiters 자동 인식 + 일반 페이지 AI 추출은 기술적으로 탄탄한 차별점.
- 무료 체험(맞춤 이력서 2개, 카드 등록 불필요) → $7.99/월 구조는 이 시장에 적절한 가격.

약점:
- 랜딩은 "전 세계 공고 검색 플랫폼"처럼 보이지만, 실제 제품은 "내가 등록한 회사의 채용페이지를 수집하는 도구". 이 기대-현실 갭이 신규 유저 이탈의 최대 원인이 될 것.
- 첫 가입 직후 "빈 화면"에 떨어지는 콜드 스타트 문제.

## 2. 여정별 UX 이슈 (우선순위순)

### 🔴 P0 — 홍보 시작 전 반드시 수정

**2-1. 랜딩의 가짜 통계 노출** — `src/lib/matchda/i18n.ts:58`
- "12,000+ 활동 중인 사용자", "45개국+", "81% 평균 매칭률", "3분 평균 번역 시간"이 목업 수치 그대로 실서비스 랜딩에 노출 중.
- 커뮤니티 홍보 시 누군가 "12,000명이라는데 후기가 왜 없죠?"라고 묻는 순간 신뢰가 무너진다. 허위·과장 광고(표시광고법) 리스크도 있음.
- 수정안: 수치 밴드를 제거하거나, 검증 가능한 사실로 교체 — "Greenhouse·Lever 등 4개 ATS 자동 인식", "가입 후 3분 안에 영어 이력서 완성" 같은 기능 기반 문구.

**2-2. 히어로 검색바의 기대-현실 갭** — `SplitHero` / `heroB.popularChips`
- 랜딩 검색바("직무, 회사 또는 기술 검색" + "전체 국가" 셀렉트 + 인기 검색 "베를린 · 백엔드, 싱가포르 PM")는 글로벌 공고 DB가 있는 것처럼 보임.
- 실제로는 가입 → `/discover?q=검색어`로 넘어가 **내가 등록한 소스에서 수집된 공고를 필터링**할 뿐. 신규 유저는 공고가 0건이라 검색 결과가 반드시 빈 화면.
- 수정안: (a) 검색바를 "관심 회사 채용페이지 URL을 붙여넣으세요" 컨셉으로 바꾸거나, (b) 공유 공고 풀(`jobs` 테이블)을 비로그인에게도 읽기 전용으로 보여줘 검색이 실제로 동작하게 하기. (b)가 SEO·전환에 훨씬 유리.

**2-3. 가입 직후 온보딩으로 안 데려감** — `auth/callback/route.ts`, `LoginForm.tsx:57`
- 로그인/가입 완료 → 무조건 `/discover` 이동. 프로필(이력서)이 없는 상태라 매칭 점수가 전부 무의미하고, 안내는 상단 얇은 배너 하나뿐.
- 수정안: 콜백/로그인 성공 시 `onboarding_completed`가 false면 `/onboarding`으로 리다이렉트. 핵심 마법(한국어 → 영어 이력서)을 가입 3분 안에 경험시키는 게 활성화의 전부다.

**2-4. 온보딩에 "이력서 업로드로 건너뛰기"가 없음** — `src/app/onboarding/`
- 이미 이력서 파일이 있는 사람(대부분)도 8단계 채팅으로 학력·경력을 다시 타이핑해야 함. PDF/DOCX 파싱 기능(`ResumeStudio` 업로드 → AI 자동 채움)이 이미 있는데 온보딩에서는 노출 안 됨.
- 수정안: 온보딩 첫 화면에 "📄 이력서 파일로 시작하기 (PDF·DOCX)" / "💬 채팅으로 작성하기" 2택 제공. 업로드 시 파싱 결과를 채팅 답변에 프리필하고 확인만 받기.

### 🟠 P1 — 초기 유저 경험 개선

**2-5. 이메일 인증 왕복 마찰**
- 이메일 가입 → "확인 메일을 보냈어요" → 메일 인증 → **다시 돌아와서 직접 로그인**. 이 왕복에서 상당수가 이탈한다.
- 개발자용 에러 문구가 유저에게 노출: `LoginForm.tsx:55` "…Supabase에서 이메일 인증을 비활성화해주세요" → 유저용 문구로 교체 필요.
- 수정안: 인증 링크 클릭 시 자동 로그인 + 온보딩 진입(Supabase confirm redirect 활용). 초기에는 Google 로그인을 더 크게 밀어도 됨.

**2-6. 정보 구조(IA) 용어 혼란**
- 사이드바 "내 이력서"를 누르면 페이지 제목은 "이력서 스튜디오"이고, 그 아래 "매칭 설정"까지 있음. "잡 탐색" 안에는 "내 채용페이지 수집 공고" vs "전체 수집 공고" 두 목록이 있는데 차이를 설명해주지 않음.
- "관리 보내기"라는 버튼명은 무슨 일이 일어나는지 예측 불가 (실제: 지원 현황 보드에 추가 + JD 분석 + AI 매칭).
- 수정안: "지원 목록에 추가" 같은 결과 중심 이름으로 변경, 두 공고 목록에 1줄 설명 툴팁 추가, 메뉴명과 페이지 제목 통일.

**2-7. 수집(스크래핑) 대기 UX**
- 수집이 최대 300초까지 걸릴 수 있는데(`discover/page.tsx maxDuration=300`) 피드백은 버튼의 "수집 중…" 텍스트뿐. 유저가 새로고침하거나 이탈하면 결과를 놓침.
- 수정안: 단계별 진행 표시("페이지 접속 → 공고 추출 → AI 채점"), 완료 시 토스트 + 결과 요약. 장기적으로는 백그라운드 잡 + 완료 알림.

**2-8. 빈 상태(콜드 스타트) 가이드 부족**
- `/discover` 첫 진입 시 폼·프리셋·빈 목록이 나열될 뿐, "무엇부터 해야 하는지" 순서가 없음.
- 수정안: 공고 0건일 때 3단계 체크리스트 히어로 노출 — ① 이력서 완성 ② 관심 회사 등록(또는 프리셋 클릭) ③ 첫 매칭 확인. 프리셋 기업 원클릭 수집은 이미 훌륭하니 이걸 첫 화면 주인공으로.

### 🟡 P2 — 다듬기

- **디버그 코드 잔존**: `LoginForm.tsx:47` `localStorage.setItem('__login_debug__', …)` — 제거.
- **랜딩 쇼케이스의 실명 같은 예시**: "김지민" 사례가 실제 후기처럼 보임 — "예시 화면" 라벨 필요. 라이브 점검에서 `matchda-mark.png` 이미지 로드 실패 의심도 확인 필요.
- **무료 한도 에러 문구**: `actions.ts:198` "(요금제 페이지 /pricing)" — 경로 텍스트 대신 실제 링크/버튼으로.
- **포지셔닝 불일치**: 프로젝트는 호주/NZ IT 중심인데 랜딩은 "45개국, 베를린, 싱가포르". 초기에는 "호주·뉴질랜드 IT 취업"으로 좁히는 게 홍보 타겟팅과 SEO 모두에 유리. 잘 되면 확장.
- **온보딩 채팅 뒤로가기 없음**: 이전 답변을 수정하려면 끝까지 간 뒤 "다시 작성"해야 함 — 단계 back 버튼 추가.
- **워크스페이스 진입 경로**: 사이드바에 워크스페이스 메뉴가 없어 공고 카드를 통해서만 진입 가능 — 의도라면 OK, 다만 처음 온 유저에게 "공고를 누르면 워크스페이스가 열려요" 힌트가 있으면 좋음.

### 잘 만든 것 (유지)

- 온보딩 채팅의 진행바 + localStorage 임시저장 + "다시 작성" 프리필
- 기가입 이메일 2중 방어와 에러 안에서 바로 "로그인하기/비밀번호 찾기" 제공 (`LoginForm.tsx:167`)
- `ConnectResumeGate` — 이력서 없는 유저에게 가짜 목업 대신 연결 안내를 보여주는 결정
- 이력서 스튜디오의 "원본 수정됨 → 영어로 동기화" 상태 표시, PDF/DOCX 다운로드
- 프리셋 기업 원클릭 수집, 무료 플랜의 명확한 한도(소스 5개·맞춤 이력서 2개)

## 3. 홍보 전략 & 문구

### 전략 요약

1. **타겟을 좁혀서 시작**: "해외 취업" 전체가 아니라 **"호주·뉴질랜드 IT 취업 준비자"**로. 오픈채팅방도 "호주 워홀/취업", "호주 개발자", "뉴질랜드 이민" 방이 밀도가 높다.
2. **메이커 스토리로 접근**: 커뮤니티(디스콰이엇, GeekNews, 커리어리, OKKY)는 광고를 싫어하고 "직접 만든 이야기"를 좋아한다. "제가 호주 취업 준비하면서 만든 도구"라는 1인칭 서사가 가장 강력.
3. **수치 약속 금지**: P0 이슈와 동일 — "12,000명" 같은 문구는 절대 사용하지 말고, 기능과 경험만 이야기한다.
4. **각 채널 규칙 준수**: 오픈채팅방은 방장에게 먼저 허락을 구하고 올리기. 무단 홍보는 강퇴 + 브랜드 이미지 훼손.

### A. 카카오 오픈채팅방용 (짧은 버전)

> 호주/뉴질랜드 취업 준비하시는 분들께 도움될까 해서 공유합니다 🙂
>
> 저도 해외 취업 준비하면서 매번 (1) 한국어 이력서를 영어로 번역하고 (2) 공고마다 이력서 고치고 (3) 커버레터 쓰고 (4) 어디 지원했는지 스프레드시트로 관리하는 게 너무 번거로워서, 이걸 하나로 합친 서비스를 직접 만들었어요.
>
> **매치다 (matchda.com)**
> - 한국어로 경력 입력하면 → AI가 영어 이력서로 정리
> - 관심 회사 채용페이지 등록하면 → 새 공고 자동 수집 + 나와의 매칭 점수
> - 공고별 맞춤 이력서·커버레터 자동 생성
> - 지원 현황 칸반보드로 관리
>
> 무료로 맞춤 이력서 2개까지 만들어볼 수 있어요 (카드 등록 X).
> 써보시고 불편한 점 알려주시면 바로 고칠게요. 피드백이 제일 감사합니다 🙏

### B. 오픈채팅방용 (초단문 버전 — 공지/방장 협의 후)

> 한국어로 쓰면 영어 이력서가 나오는 호주·NZ 취업 도구 만들었어요.
> 공고 자동 수집 → 매칭 점수 → 맞춤 이력서·커버레터까지 한 번에.
> 무료 체험 가능: matchda.com (피드백 주시면 바로 반영합니다!)

### C. 커뮤니티 장문용 (디스콰이엇 / GeekNews / 클리앙 / 네이버 카페)

제목 후보:
- "호주 취업 준비하다가 열받아서 직접 만든 이력서 도구, 무료로 풀어봅니다"
- "한국어로 답하면 영어 이력서가 나옵니다 — 해외 취업 준비 올인원 툴 만든 후기"

본문:

> 해외(호주) 취업을 준비하면서 제일 시간을 많이 잡아먹은 건 코딩테스트도 면접도 아니고 **서류 노가다**였습니다.
>
> - 한국어 이력서 → 영어 번역 (DeepL 돌리고 어색한 표현 다시 수정)
> - 공고마다 요구사항이 달라서 이력서를 매번 조금씩 수정
> - 커버레터는 공고마다 새로 작성
> - 어디에 어떤 버전으로 지원했는지 스프레드시트 관리
>
> 이 과정을 자동화하려고 **매치다(matchda.com)**를 만들었습니다.
>
> **동작 방식**
> 1. 한국어로 채팅하듯 경력을 입력하면 (이력서 파일 업로드도 가능) AI가 영어 이력서로 정리합니다.
> 2. 관심 회사의 채용페이지 URL을 등록하면 새 공고를 자동으로 수집합니다. Greenhouse·Lever·Ashby·SmartRecruiters는 자동 인식되고, 일반 페이지는 AI가 공고 목록을 추출합니다.
> 3. 수집된 공고마다 내 이력서와의 매칭 점수와 이유를 보여줍니다.
> 4. 지원할 공고를 고르면 그 공고의 JD에 맞춰 이력서를 최적화하고 커버레터를 생성합니다.
> 5. 준비 → 지원 → 면접 → 오퍼, 칸반보드로 지원 현황을 관리합니다.
>
> 무료로 맞춤 이력서 2개까지 만들어볼 수 있고 카드 등록은 필요 없습니다.
> 아직 다듬는 중이라 피드백 주시면 정말 빠르게 반영하겠습니다. 특히 호주·뉴질랜드 외 지역 수요가 있는지 궁금합니다.

### D. 커리어리 / 스레드 / 링크드인용 (훅 중심)

> 해외 취업 서류 준비의 진실:
> 공고 1개 지원 = 이력서 수정 30분 + 커버레터 1시간 + 트래킹 시트 정리.
> 10개 지원하면 15시간입니다.
>
> 그래서 만들었습니다.
> 한국어로 입력 → 영어 이력서 → 공고 자동 수집 → 매칭 점수 → 공고별 맞춤 이력서·커버레터 → 칸반 관리까지.
>
> 매치다 · matchda.com · 무료로 시작
